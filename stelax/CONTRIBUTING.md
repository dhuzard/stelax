# Contributing to STELAX

Thank you for your interest in contributing to the STELAX template!

## PR Policy
1. One PR = one concern.
2. Every PR must link to exactly one issue.
3. PR title format: `[<area>] <imperative summary>`
4. PR body must use a repository PR template (`.github/PULL_REQUEST_TEMPLATE/`).
5. Direct commits to `main` are forbidden.
6. Infra/security PRs require QA + Security review.
7. Issue forms, schemas, and workflows require Orchestrator review.
8. Merge strategy: squash only.

Review `docs/status/` for current agent assignments and progress.
